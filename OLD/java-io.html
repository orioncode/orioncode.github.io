Explanation of JDK12 Source Code

JDK12 package = java.io

Bits.java

Static methods that convert byte array values to boolean, short, ints, etc. and it can also put boolean values or short or int, etc. to the byte array. It uses big-endian byte ordering which means that the most significant bits of the byte value are on the left of the bit string. For example if you have a byte array you can extract the byte value at the desired index and convert it to boolean or a short or a char, etc. If a byte value, for example, is 0 then it returns false. Otherwise true. If you want to extract the short value then it extracts 2 byte values since short data types use 2 bytes in memory. Similarly it can put values insise a byte array

BufferedInputStream.java

It represents an input stream of bytes that is manipulated by an internal volatile buffer array. It does not read the entire input stream. It loads 8KB from the stream in the buffer, it performs the desired operations and then when the buffer has been processed then it loads another 8KB and so on. Teh default buffer size is 8KB. The mark operation remembers a point in the input stream. The reset operation rereads the bytes from the current buffer that begin from the marked byte/position. The maximum buffer size is Integer.MAX_VALUE - 8 = 2,147,483,640. It keeps a count of the next available byte to be read from the original stream. It keeps the position of the next available byte to be read from the buffer.

BufferedOutputStream.java

It represents an output stream of bytes that can be written to an output stream using an internal buffer array. The default buffer size is 8KB. It keeps a count of the valid bytes that exist in the buffer. When we request to write something in the stream then it adds the bytes we just requested to the buffer. If the buffer is full then it flushes it i.e. it writes the buffer contents to the stream and it resets the count to 0 and if we write more bytes then they are put at the beginning of the buffer and so on.

BufferedReader.java

It reads characters (not bytes). It can be used to read a file, for example, using the inherent buffer by calling: new BufferedReader(new FileReader("foo.txt")). This way you wrap the BufferedReader around a FileReader (that can be costly if it is a big file). The default buffer size is 8KB.

BufferedWriter.java

It writes characters (not bytes). It can be used to write text to a file, for example, using the inherent buffer by calling: new BufferedWriter(new FileWriter("foo.txt")). This way you wrap the BufferedWriter around a FileWriter (that can be costly if it is a big file). The default buffer size is 8KB. It keeps a count of the valid bytes that exist in the buffer. When we request to write something in the stream then it adds the characters we just requested to the buffer. If the buffer is full then it flushes it i.e. it writes the buffer contents to the stream and it resets the count to 0 and if we write more characters then they are put at the beginning of the buffer and so on.
